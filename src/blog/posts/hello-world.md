Starting a blog feels a bit like opening a new terminal — you're staring at a blinking cursor, waiting for the first command.

## Why Now?

I've been meaning to write publicly for a while. Mostly to organize my thoughts, document things I learn, and maybe help someone who's searching for the same obscure error message I spent three hours debugging.

Writing forces clarity. If I can't explain something in plain sentences, I probably don't understand it as well as I think I do.

## What to Expect

Mostly technical content:

- Software engineering patterns and practices
- Project breakdowns and post-mortems
- Tools I'm using and why
- Occasionally, thoughts on the broader industry

## The Format

Short and precise. I prefer writing that gets to the point. If I can explain something in 500 words, I won't use 2000.
